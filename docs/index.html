Scout Engine — architecture

Two halves: how the data gets produced (dbt), and how the app serves it (Streamlit).

1 · data pipeline
Understat
players
→
stg_players
→
mart_player_season
incremental
dim_player_latest
+ SCD2 snapshot
train.py
scaler + k-means
mart_player_breakout
YoY output delta
Understat
matches
→
stg_matches
→
mart_team_season

mart_player_season is incremental (only reprocesses rows newer than the last fetch), and a Type-2 SCD snapshot tracks each player's team/league/position history instead of overwriting it silently.

2 · how the app serves it
Browser
→
app.py
UI only
data.py
cached loaders → DuckDB / joblib / parquet
similarity.py
on-demand k-NN per query

Cached with @st.cache_resource / @st.cache_data, keyed on each artifact file's mtime — so a retrained model on disk gets picked up without a stale in-memory cache surviving it.
